# 接线步骤 — 按已购器件逐步验证

> 目标：每接完一个模块就验证一次，避免整车一次性接完后无法定位问题。
> 所有接线、换线、插拔模块操作都在**断电状态**下进行。

---

## 0. 当前器件边界

本文只使用 `器件采购清单.md` 里已有/计划购买的器件，不假设额外 MOSFET、三极管、专用电源板或示波器。

已购/计划相关器件包括：

- LP-MSPM0G3507 LaunchPad
- WHEELTEC R3X 三轮底盘，含 2 个编码电机和 1 个 TB6612 驱动模块
- 2S 18650 电池盒，输出约 7.4V
- Micro-USB 数据线
- 杜邦线、万用表
- 5V 充电宝
- AMS1117-3.3V 模块
- MP1584 DC-DC 可调降压模块
- TCRT5000 独立小板，4Pin: `VCC/GND/DO/AO`
- MG90S 舵机 ×2
- OpenMV H7
- 405nm 可调焦激光头
- 5mm LED + 330Ω 电阻，红/蓝电源 LED + 1kΩ 电阻
- 按键、洞洞板、电容、扎带、热缩管等

## 1. 总原则

### 1.1 先分模块，后整车

推荐顺序：

```text
LaunchPad 单独烧录
→ TB6612 + 电机
→ 编码器
→ 巡线传感器
→ 舵机
→ OpenMV
→ 激光
→ 按键/状态 LED
→ 整机联调
```

不要一开始就把所有模块接上。

### 1.2 全系统必须共地

只要两个模块之间有信号线，就必须共地。

```text
LaunchPad GND
TB6612 GND
电池负极
MP1584 GND
5V 充电宝 GND
OpenMV GND
MG90S GND
TCRT5000 GND
编码器 GND
```

最终都要连通。

### 1.3 任何 7.4V 都不能直接进 MSPM0 GPIO

MSPM0 GPIO 是 3.3V 逻辑。  
禁止把电池 7.4V 接到任何 GPIO、`3.3V`、`5V`、传感器信号脚。

## 2. 当前阶段：电机与 TB6612 验证

你们目前已经连接了：

```text
MSPM0G3507
TB6612
电机
2S 18650 电池
```

先完成这一节，再接别的模块。

### 2.1 电机验证接线

| LaunchPad | TB6612 | 说明 |
|---|---|---|
| `PB0` | `AIN1` | 左电机方向 1 |
| `PB1` | `AIN2` | 左电机方向 2 |
| `PB2` | `BIN1` | 右电机方向 1 |
| `PB3` | `BIN2` | 右电机方向 2 |
| `PB4` | `STBY` | TB6612 使能，HIGH=工作 |
| `PA12` | `PWMA` | 左电机 PWM |
| `PA13` | `PWMB` | 右电机 PWM |
| `LaunchPad GND` | `TB6612 GND` | 共地 |
| `LaunchPad 3.3V` | `TB6612 VCC` | 逻辑电源 |
| `电池 7.4V+` | `TB6612 VM` | 电机电源 |
| `电池 7.4V-` | `TB6612 GND` | 电机电源地 |

电机输出：

```text
TB6612 AO1/AO2 → 左电机
TB6612 BO1/BO2 → 右电机
```

如果方向反了，交换该电机的两根线。

### 2.2 上电前检查

断电状态下检查：

- [ ] 电池正负没有接反
- [ ] `TB6612 VM` 没有接到 LaunchPad
- [ ] `TB6612 VCC` 是 3.3V，不是 7.4V
- [ ] `LaunchPad GND`、`TB6612 GND`、`电池负极` 连通
- [ ] 电机悬空，轮子不接触桌面

上电后用万用表测：

| 测量点 | 正常值 |
|---|---|
| `TB6612 VM-GND` | 约 7.0V-8.4V |
| `TB6612 VCC-GND` | 约 3.3V |
| `PB4/STBY-GND` | 程序运行后约 3.3V |
| `PA12/PWMA-GND` | 有高电平或 PWM |
| `PA13/PWMB-GND` | 有高电平或 PWM |

### 2.3 手动硬件验证：不用 PWM

这一步用于确认电池、TB6612、电机本体没有问题。

断电后接线：

```text
TB6612 STBY → 3.3V
TB6612 PWMA → 3.3V
TB6612 PWMB → 3.3V
```

左电机正转：

```text
AIN1 → 3.3V
AIN2 → GND
```

右电机正转：

```text
BIN1 → 3.3V
BIN2 → GND
```

如果这样电机能转，说明：

```text
电池 OK
TB6612 OK
电机 OK
```

如果这样也不转，先不要查代码，优先查 `VM/VCC/GND/STBY/PWMA/PWMB`。

### 2.4 GPIO 满速验证

这一步用于确认 MSPM0 的 GPIO 能控制 TB6612。

使用已保存的测试文件：

```text
C:\Users\1\Desktop\motor_gpio_fullspeed_test.c
```

现象：

```text
PA12/PA13 被当作 GPIO 高电平
PWMA/PWMB 得到 3.3V
双电机满速正转
```

如果这一步能转，说明 `PA12/PA13 → PWMA/PWMB` 接线没有问题。

### 2.5 PWM 调速验证

PWM 版需要注意三个关键点：

```text
PA12 = IOMUX_PINCM34_PF_TIMG0_CCP0
PA13 = IOMUX_PINCM35_PF_TIMG0_CCP1
TIMG0 配置完成后必须调用 DL_TimerG_startCounter(TIMG0)
```

常见错误：

```text
只配置了 TIMG0 PWM，但没有 start counter
→ PA12/PA13 没有 PWM
→ PWMA/PWMB 等价于 0%
→ 电机不转
```

验证通过的现象：

```text
30% 占空比转 3 秒
60% 占空比转 3 秒
90% 占空比转 3 秒
停止 2 秒
循环
```

## 3. 电源规划

### 3.1 调试阶段推荐供电

调试时最稳的供电方式：

```text
LaunchPad → 电脑 USB 供电
TB6612 VM → 2S 18650 电池 7.4V
TB6612 VCC → LaunchPad 3.3V
所有 GND 共地
```

这适合下载程序、单独验证电机、编码器、传感器。

### 3.2 后续整车供电

根据已购器件，推荐：

```text
控制回路:
2S 18650 → 开关1 → TB6612 VM
LaunchPad 调试时仍用 USB
如需脱离电脑运行，再考虑用 AMS1117-3.3V 给 3.3V 负载供电

瞄准回路:
5V 充电宝 → 开关2 → 舵机/OpenMV
```

MP1584 的定位：

```text
有 5V 充电宝时，优先用充电宝给舵机/OpenMV 供电。
不用充电宝时，才用 MP1584 将 7.4V 降到 5.0V。
```

### 3.3 MP1584 预调

如果使用 MP1584：

```text
电池 7.4V+ → MP1584 IN+
电池 7.4V- → MP1584 IN-
```

先不接舵机和 OpenMV，只接万用表。

- 调节电位器到 `5.0V ±0.1V`
- 调好后断电
- 再接负载

### 3.4 AMS1117 注意事项

AMS1117-3.3V 可用于给 3.3V 小负载供电，但不要把它当作大电流电源。

禁止：

```text
7.4V → LaunchPad 5V
7.4V → LaunchPad 3.3V
7.4V → 任意 GPIO
```

## 4. 编码器接线

### 4.1 接线表

| 编码器 | LaunchPad |
|---|---|
| 左轮 A 相 | `PC0` |
| 左轮 B 相 | `PC1` |
| 右轮 A 相 | `PC2` |
| 右轮 B 相 | `PC3` |
| 编码器 VCC | `3.3V` |
| 编码器 GND | `GND` |

颜色可能随厂家变化，不要只按颜色接，优先看卖家说明或线标。

### 4.2 验证

先不让电机自动转，手动慢转轮子：

- A/B 相对 GND 应在 `0V` 和 `3.3V` 之间变化
- 用程序计数时，输出轴一圈理论边沿数约：

```text
500线 × 30减速比 × 4倍频 = 60000 边沿/轮圈
```

如果只用单 A 相双沿，计数会低于 4 倍频结果，按代码定义确认。

## 5. 巡线传感器接线

### 5.1 使用已购 TCRT5000 小板

已购模块按 4Pin 小板处理：

```text
VCC
GND
DO
AO
```

当前整车无冲突方案优先使用 `DO` 数字输出。  
`AO` 暂时不接，后续如果改模拟巡线再重新分配 ADC 引脚。

### 5.2 数字巡线接线

| 传感器 | LaunchPad |
|---|---|
| S1 最左 | `PA2` |
| S2 | `PA5` |
| S3 | `PA6` |
| S4 中间 | `PA7` |
| S5 | `PA8` |
| S6 | `PB9` |
| S7 最右 | `PB10` |

每个模块：

```text
VCC → LaunchPad 3.3V
GND → LaunchPad GND
DO  → 对应 GPIO
AO  → 暂不接
```

### 5.3 安装

```text
车头底部，从左到右:
[S1] [S2] [S3] [S4] [S5] [S6] [S7]
```

建议：

- 间距：1.0cm-1.5cm
- 离地：3mm-5mm
- 尽量一条直线安装
- 固定后再调阈值

### 5.4 阈值调节

用万用表测 `DO-GND`：

```text
白底：LOW
黑线：HIGH
```

如果相反，代码里取反即可，但 7 个传感器必须一致。

## 6. 舵机云台接线

### 6.1 供电

MG90S 不从 LaunchPad 5V 取电。

推荐：

```text
MG90S 红线 → 5V 充电宝输出 或 MP1584 5V 输出
MG90S 棕线 → 公共 GND
MG90S 橙线 → MSPM0 PWM 信号
```

### 6.2 信号线

| 舵机 | 信号引脚 |
|---|---|
| 水平 X | `PA3` |
| 俯仰 Y | `PA4` |

MSPM0 的 3.3V PWM 通常可以被 MG90S 识别。

### 6.3 验证

先只接一个舵机：

```text
1500us → 中位
1000us/2000us → 左右或上下偏转
```

确认不抖、不重启、不发热后，再接第二个。

## 7. OpenMV 接线

### 7.1 供电

```text
OpenMV VIN → 5V 充电宝 或 MP1584 5V
OpenMV GND → 公共 GND
```

不要用 LaunchPad 3.3V 直接供 OpenMV。

### 7.2 UART

```text
OpenMV P4 (RX) ← PA10 (MSPM0 TX)
OpenMV P5 (TX) → PA11 (MSPM0 RX)
```

注意 TX/RX 交叉：

```text
MSPM0 TX 接 OpenMV RX
MSPM0 RX 接 OpenMV TX
```

如果使用 UART0 和 OpenMV 通信，调试 printf 不要同时占用 UART0。

## 8. 激光模块接线

### 8.1 先确认激光头类型

已购 405nm 激光头大概率是两线模块：

```text
VCC
GND
```

如果只有两根线，**不能直接用 PA20 给激光供电**。  
MSPM0 GPIO 不能承担激光电流，直接接有烧 GPIO 风险。

### 8.2 当前已购器件下的安全接法

没有额外 MOSFET/三极管时，先用手动供电验证：

```text
激光 VCC → 对应规格电源
激光 GND → GND
```

用开关或插拔供电控制亮灭。

### 8.3 只有带 TTL/EN 信号脚的激光模块才接 PA20

如果激光模块明确有：

```text
VCC
GND
TTL/EN/S
```

才可以：

```text
TTL/EN/S → PA20
VCC → 模块规格要求的 3.3V 或 5V
GND → 公共 GND
```

否则 `PA20 → 激光` 这条线暂时不要接。

## 9. 按键与状态 LED

### 9.1 按键

独立按键用于模式和启动：

| 功能 | 引脚 |
|---|---|
| MODE | `PB8` |
| START | `PB11` |

接法：

```text
按键一端 → GPIO
按键另一端 → GND
```

代码中使用内部上拉：

```text
松开 = HIGH
按下 = LOW
```

四脚轻触按键同侧两脚通常内部相连。接线前用万用表通断档确认哪两脚常通。

### 9.2 状态 LED

BOM 里状态 LED 配的是 330Ω 电阻，接法：

```text
PA15 → 330Ω → LED 长脚(+)
LED 短脚(-) → GND
```

电源指示 LED：

```text
控制回路电源 LED → 1kΩ → GND
瞄准回路电源 LED → 1kΩ → GND
```

## 11. 当前推荐下一步

你们已经完成电机 PWM 验证，下一步建议按顺序做：

```text
1. 保存电机 GPIO 满速版和 PWM 调速版测试程序
2. 接编码器
3. 写/烧录编码器计数测试程序
4. 验证手转轮子能计数
5. 再做闭环速度控制
```

不要在编码器验证前接巡线、舵机和 OpenMV。
