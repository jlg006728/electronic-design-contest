# OpenMV / K230 视觉模块选型对比

## 概述

E题瞄准模块的核心挑战是：从50cm外识别靶心（直径0.1cm），驱动云台瞄准。需要可靠的机器视觉方案。

## 方案一: OpenMV Cam

OpenMV 是专为电赛设计的嵌入式机器视觉模块，MicroPython 编程，生态成熟。

### 硬件规格

| 参数 | OpenMV M7 | OpenMV H7 | OpenMV RT1062 |
|------|-----------|-----------|---------------|
| MCU | STM32F765VI | STM32H743VI | NXP RT1062 |
| 架构 | Cortex-M7 | Cortex-M7 | Cortex-M7 |
| 主频 | 216 MHz | 480 MHz | **600 MHz** |
| RAM | 512 KB | 1 MB | 32 MB |
| Flash | 2 MB | 2 MB | 16 MB |
| Sensor | OV7725 | OV5640 | OV5640 |
| 最大分辨率 | 640x480 | 2592x1944 | 2592x1944 |
| FPS @ VGA | 60 | 60 | 120+ |
| 价格 | ~$55 | ~$65 | ~$85 |

### OpenMV 机器视觉能力

- 颜色追踪 (同时 32 种颜色)
- AprilTag 追踪 (~12 FPS @ 160x120)
- 边缘检测 (Canny + Hough 直线/圆检测)
- 模板匹配
- 人脸检测 (Haar Cascade)
- QR码/条形码解码

### E题适用性

| 项目 | 评估 |
|------|------|
| 靶心识别 | 红色圆点检测 + 同心圆识别 |
| 处理速度 | VGA @ 60FPS, 满足实时控制 |
| 开发难度 | MicroPython, 简单 |
| 电赛生态 | 有大量电赛参考代码 |
| 体积 | 45x36x30mm, 适合小车 |

## 方案二: K230 (嘉楠堪智)

新一代 RISC-V AI 视觉芯片，6 TOPS NPU 算力。

### 硬件规格

| 参数 | 数值 |
|------|------|
| CPU | 双核 XuanTie C908 RISC-V (1.6GHz + 800MHz) |
| NPU | 第三代 KPU, **6 TOPS** (INT8) |
| 内存 | 标准 1GB LPDDR4 |
| VPU | H.264/H.265 4K编解码 |
| CSI | 3路 MIPI CSI |
| DSI | 1路 MIPI DSI (1920x1080) |

### AI 推理性能

| 模型 | 帧率 (INT8) |
|------|------------|
| ResNet-50 | >= 85 fps |
| MobileNetV2 | >= 670 fps |
| YOLOv5s | >= 38 fps |

### E题适用性

| 项目 | 评估 |
|------|------|
| 算力 | 过剩 (6 TOPS 远超需求) |
| 开发难度 | 较高, RISC-V 生态尚在完善 |
| 功耗 | 较高 (需散热) |
| 体积 | 较大 (典型开发板 65x40mm+) |
| 电赛生态 | 较新, 参考资料少 |

## 选型建议

| 方案 | 推荐场景 |
|------|----------|
| **OpenMV H7** | 电赛首选 - 成熟可靠, Python开发快, 生态好 |
| OpenMV M7 | 预算有限 - 基本够用, 颜色追踪+FPS足够 |
| K230 | 有经验/追求极致 - AI能力过剩, 但开发复杂 |

## 备选: HUSKYLENS 2

一体化 AI 视觉传感器:
- 内置 K230 + 2.4寸触摸屏
- 预置 20+ AI模型, 免训练
- 学习模式: 一键学习目标颜色/物体
- 缺点: 灵活性不如 OpenMV

## 补充: 普通摄像头 + MCU 处理

部分获奖方案使用:
- 海康机器人相机 + Jetson Orin NX
- USB 摄像头 + PC 处理 (不符合电赛规定, 禁用PC)
- ESP32-CAM (分辨率低, 不推荐)

> **结论**: E题推荐使用 **OpenMV Cam H7** 作为视觉模块。
